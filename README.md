# Smart Desktop Organizer (Умный Организатор Рабочего Стола)

## Описание проекта

Smart Desktop Organizer - это приложение для Windows, разработанное для помощи пользователям в поддержании порядка на рабочем столе. Оно автоматизирует рутинные задачи по очистке недействительных ярлыков и пустых папок, а также предлагает интеллектуальную сортировку элементов рабочего стола на основе их типа и пользовательских настроек. Программа разработана на C++ с использованием MFC для графического интерфейса и CMake для сборки.

## Основные возможности

Программа предоставляет две основные функции: "Проверка рабочего стола" и "Сортировка рабочего стола", а также окно настроек для кастомизации поведения.

### 1. Проверка рабочего стола

Эта функция сканирует ваш рабочий стол на наличие:

*   **Недействительных ярлыков:** Ярлыки (.lnk файлы), которые ссылаются на несуществующие файлы или папки.
*   **Пустых папок:** Папки, не содержащие файлов или других подпапок.

После сканирования программа отображает список найденных элементов. Вы можете выбрать, какие из них удалить. Удаление может производиться как в Корзину, так и безвозвратно (настраивается в "Настройках").

### 2. Сортировка рабочего стола

Эта функция организует элементы на вашем рабочем столе по следующим правилам:

*   **Сбор информации:** Программа определяет текущее разрешение экрана, DPI масштабирование и список всех элементов на рабочем столе с их текущими координатами и датой последнего изменения.
*   **Классификация элементов:**
    *   Ярлыки (.lnk) и исполняемые файлы (.exe) классифицируются как "Игра" или "Программа".
    *   Классификация происходит с помощью:
        1.  Поиска информации в Google по имени файла/ярлыка (можно отключить в настройках).
        2.  Поиска по имени родительской папки, если имя файла неоднозначно.
        3.  Локальных правил на основе расширений файлов (например, `.txt`, `.log` - текстовые файлы; `.jpg`, `.png` - изображения) и ключевых слов в имени.
    *   Результаты веб-классификации кэшируются для ускорения последующих сортировок.
*   **Логика размещения:**
    *   **Папки:** Перемещаются в правый нижний угол экрана, располагаются справа налево, рядами снизу вверх.
    *   **Текстовые файлы (.txt, .log, .md и др.):** Размещаются над папками, по тем же правилам (справа налево, ряды снизу вверх).
    *   **Ярлыки/файлы Игр:** Перемещаются в правый верхний угол, располагаются справа налево, рядами сверху вниз.
    *   **Ярлыки/файлы Программ:** Перемещаются в левую часть экрана, сортируются по дате изменения (от новых к старым), располагаются сверху вниз, рядами слева направо.
    *   **Остальные/Неклассифицированные элементы:** Размещаются в центральной части экрана (или в другой настроенной области), сортируются по дате изменения.
    *   Все элементы внутри своих категориальных групп предварительно сортируются по дате последнего изменения (от новых к старым) перед размещением.

### 3. Настройки

Окно настроек позволяет:

*   Включать или отключать веб-классификатор (поиск в Google).
*   Очищать кэш результатов веб-классификации.
*   Выбирать способ удаления элементов при "Проверке рабочего стола" (в Корзину или безвозвратно).
*   Управлять списком исключенных путей (файлы и папки, которые не будут затрагиваться функциями проверки и сортировки).

## Требования к системе

*   **Операционная система:** Windows 10 (x64) или Windows 11 (x64).
*   **Для сборки из исходного кода:**
    *   Visual Studio 2022 (рекомендуется Community Edition или выше).
    *   Рабочая нагрузка "Разработка классических приложений на C++" (Desktop development with C++).
    *   Компоненты CMake для Windows, поставляемые с Visual Studio.
    *   Интернет-соединение (для первоначальной загрузки зависимостей CMake и для функции веб-классификации).

## Установка и сборка (из исходного кода в Visual Studio 2022)

1.  **Клонирование репозитория:**
    *   Откройте Visual Studio 2022.
    *   На стартовом экране выберите "Клонировать репозиторий" (Clone a repository).
    *   Введите URL репозитория GitHub и выберите локальный путь для сохранения. Нажмите "Клонировать".
    *   Или используйте Git в командной строке: `git clone <URL_репозитория>`

2.  **Открытие проекта в Visual Studio:**
    *   После клонирования Visual Studio может автоматически предложить открыть проект.
    *   Если нет, выберите "Файл" (File) -> "Открыть" (Open) -> "Папка..." (Folder...).
    *   Перейдите в папку, куда вы клонировали репозиторий, и выберите ее. Visual Studio автоматически обнаружит `CMakeLists.txt`.

3.  **Конфигурация CMake:**
    *   Visual Studio автоматически начнет процесс конфигурации CMake. Это может занять некоторое время, так как будут загружены зависимости (например, библиотека nlohmann_json).
    *   Вывод процесса CMake можно увидеть в окне "Вывод" (Output), выбрав "CMake" в выпадающем списке "Показать вывод от:" (Show output from:).
    *   Если конфигурация не началась автоматически:
        *   В "Обозревателе решений" (Solution Explorer) убедитесь, что вы находитесь в режиме просмотра папок (Solutions and Folders / Folder View).
        *   Щелкните правой кнопкой мыши по файлу `CMakeLists.txt` и выберите "Конфигурировать кэш" (Configure Cache) или "Создать CMake кэш" (Generate Cache).

4.  **Выбор конфигурации сборки:**
    *   На панели инструментов Visual Studio выберите конфигурацию сборки (например, `x64-Debug` или `x64-Release`). Рекомендуется `x64-Debug` для первоначальной проверки.

5.  **Выбор элемента автозагрузки:**
    *   На панели инструментов Visual Studio в выпадающем списке "Выбор элемента автозагрузки" (Select Startup Item) выберите `SmartDesktopOrganizer.exe`.

6.  **Сборка проекта:**
    *   В главном меню выберите "Сборка" (Build) -> "Собрать все" (Build All).
    *   Или щелкните правой кнопкой мыши по `CMakeLists.txt` в "Обозревателе решений" и выберите "Собрать" (Build).

7.  **Запуск приложения:**
    *   После успешной сборки исполняемый файл `SmartDesktopOrganizer.exe` будет находиться в подпапке `build` (или `out/build/...` в зависимости от настроек VS CMake), например, `build/src/Debug/SmartDesktopOrganizer.exe` или `build/Debug/SmartDesktopOrganizer.exe`.
    *   Вы можете запустить его напрямую из Visual Studio, нажав зеленую кнопку "Play" (треугольник) с выбранным `SmartDesktopOrganizer.exe` в качестве элемента автозагрузки.

## Использование программы

После запуска приложения вы увидите главное окно с тремя кнопками:

*   **"Проверить рабочий стол":**
    1.  Нажмите эту кнопку. Начнется сканирование рабочего стола.
    2.  Появится диалоговое окно "Результаты проверки рабочего стола", отображающее списки недействительных ярлыков и пустых папок. Элементы по умолчанию будут отмечены для удаления.
    3.  Вы можете снять галочки с элементов, которые не хотите удалять. Используйте кнопки "Выбрать все" / "Снять выделение со всех" для каждой категории.
    4.  Нажмите "Удалить выбранное".
    5.  Появится диалоговое окно с подтверждением удаления, указывающее, будут ли файлы перемещены в Корзину или удалены безвозвратно (в зависимости от настроек). Подтвердите действие.
    6.  Программа выполнит удаление.

*   **"Сортировать рабочий стол":**
    1.  Нажмите эту кнопку.
    2.  Программа начнет сбор информации, классификацию элементов (может занять время, если включен веб-классификатор и много новых элементов) и применение новой раскладки.
    3.  По завершении появится сообщение об окончании сортировки. Иконки на рабочем столе будут перестроены.

*   **"Настройки":**
    1.  Нажмите эту кнопку, чтобы открыть окно настроек.
    2.  **Веб-классификатор:** Установите/снимите галочку "Включить веб-классификатор".
    3.  **Очистить кэш классификатора:** Нажмите, чтобы удалить сохраненные результаты веб-классификации. Потребуется подтверждение.
    4.  **Удаление:** Установите/снимите галочку "Использовать Корзину (иначе безвозвратное удаление)".
    5.  **Исключенные пути:**
        *   Чтобы добавить путь в исключения, введите его в текстовое поле и нажмите "Добавить путь". Можно также нажать "Обзор..." для выбора папки.
        *   Чтобы удалить путь, выберите его в списке и нажмите "Удалить выбранный путь".
    6.  Нажмите "OK" для сохранения настроек или "Отмена" для выхода без сохранения. Настройки сохраняются автоматически при изменении.

## Файлы конфигурации и логирования

Программа создает следующие файлы в той же папке, где находится исполняемый файл (`SmartDesktopOrganizer.exe`):

*   `settings.json`: Хранит пользовательские настройки (состояние веб-классификатора, использование Корзины, список исключений).
*   `classification_cache.json`: Кэширует результаты веб-классификации для ускорения повторных сортировок и уменьшения количества запросов к Google.
*   `SmartDesktopOrganizer.log`: Подробный лог-файл всех операций программы, включая сканирование, классификацию, сортировку, ошибки. Полезен для отладки и понимания поведения программы.

## Известные проблемы и ограничения

*   **Точность веб-классификатора:** Классификация игр и программ с помощью поиска в Google может быть не всегда точной и зависит от результатов поиска и ключевых слов. Улучшенная версия использует анализ заголовков страниц, но все равно не идеальна.
*   **Производительность:** Сканирование и сортировка (особенно с веб-классификацией) на рабочих столах с очень большим количеством элементов могут занимать продолжительное время. Операции выполняются синхронно в основном потоке GUI.
*   **Взаимодействие с рабочим столом:** Перемещение иконок осуществляется через взаимодействие с системным компонентом `SysListView32`. Это может быть чувствительно к темам Windows или сторонним утилитам, изменяющим рабочий стол.
*   **Определение специальных системных папок:** Текущий механизм определения специальных папок для исключения из проверки на пустоту основан на именах и может быть неполным.
